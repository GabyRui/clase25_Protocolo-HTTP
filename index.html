¿Cómo se transfiere la informacion basado en el protocolo http?

Se puede transferir información en Javascript a partir del protocolo HTTP por medio de distintas técnicas y API disponibles. Las más aplicadas son las siguientes:
-XML HttpRequest: Permite realizar solicitudes HTTP asíncronas. Es decir, enviar y recibir datos entre el cliente y el servidor sin recargar la página. Es la API más antigua.
-Fetch: Es más flexible y de fácil uso para la realización de solicitudes HTTP. Se utilizan promesas y permite enviar solicitudes GET, POST, PUT, DELETE. Es más utilizada en sitiosn web modernos.
-Axios: Facilita más las solicitudes HTTP, y permite un mejor manejor de errores por medio de un API de un uso más sencillo. También, permite realizar solicitudes GET, POST, PUT, DELETE. También, tiene una estructura más eficiente para los desarrolladores.

¿Qué son los verbos HTTP y cuales usamos?

También son llamados métodos HTTP, y hacen referencia a las acciones utilizadas para nombrar las operaciones que se deben realizar en una parte del sitio web, y definen el tipo de solicitud que se está realizando.
-GET (obtener información): Permite solicitar recursos del servidor y obtener información sin alterar los datos. Un ejemplo común, es al momento de cargar un sitio web, ya que el navegador emite solicitudes, para obtener toda la información contenida.
-POST (crear información): Envía datos al servidor para que sean procesados, como por ejemplo, los formularios.
-PUT (actualizar información): Actualiza un recurso que se encuentra en el servidor. Mediante una solicitud PUT se pueden enviar datos al servidor para reemplazart o actualizar el recurso según los datos proporcionados.
-DELETE (eliminar información): Elimina un recurso del servidor de forma permanente.

¿Qué es un crud?

Hace referencia a las operaciones básicas que se realizan en un sistema de gestión de datos. Las 4 operaciones CRUD se refiere a las siguientes acciones:
-C (Create): La operación permite la creación de nuevos datos en la base de almacenamiento.
-R (Read): La lectura quiere decir "recuperación de datos ya existentes en la base". Lee información específica según criterios a detalle, como por ejemplo, buscar un registro.
-U (Update): Operación de actualización que trabaja en la modificación de los datos existentes en la base. Quiere decir que los valores de un registro cambiarán.
-D (Delete): Elimina datos existentes de la base de almacenamiento.
CRUD es esencial ya que incorpora las operaciones más comunmente realizadas por los usuarios. Los sitios web y las aplicaciones implementan CRUD para que los usuarios tengan una interacción efectiva y práctica.

¿Qué es asincronía y sincronia?

Son conceptos de esencial conocimiento en Javascript, ya que hace referencia a cómo se procesan las tareas en un programa. El concepto de cada uno es el siguiente.
-Sincronía: Las tareas son trabajadas en secuencia, es decir, una inmediatamente después de la otra. La tarea indicada debe completarse para pasar a la siguiente.
-Asincronía: En este caso, la ejecución de las tareas se pueden dar en paralelo, y sin necesidad de que la tarea establecida anteriormente sea completada. Se utilizan Callbacks, Promises y Async/Await para el manejo de la asincronía.

¿Cómo maneja la asincronia js?

Se utilizan mecanismos para la ejecución de tareas. Estos son los siguientes
-Callbacks: Es una forma antigua de trabajar la asincronía en Javascript y son funciones que sirven de argumentos para otras funciones, y se procesan una vez se complete una operación asincrónica.
-Promise: Es una forma más actualizada de manejar la asincronía, ya que una promesa hace referencia a un valor que puede estar disponible en cualquier momento. Tiene una estructura más eficiente y legible.
-Async/Await: Este método simplifica más el trabajo de asincronía en Javascript, y es aun más legible y con una estructura más simple.
